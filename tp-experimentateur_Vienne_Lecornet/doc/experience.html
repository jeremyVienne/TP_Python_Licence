experience module¶

experience module : Module to manages markers and experiences

author:FIL - IEEA - Univ. Lille1.fr <http://portail.fil.univ-lille1.fr>
date:2015, december

A marker is a represented as a String. An exeperience is simply a list of positive markers selected from a list of markers.

experience.compare(m1, m2)[source]¶

Compares markers m1 and m2.

Parameters:
  • m1 (String) – The first marker
  • m2 (String) – The first marker
Returns:

-1, 0 or 1 resp. if m1 < m2, m1 = m2 or m1 > m2
Return type:

int

 
>>> compare("m45","m234")
-1
>>> compare("m45","m45")
0
>>> compare("m45","m4")
1
experience.experience(p, m)[source]¶

Produces the results of an experience on p positive markers among the list of m markers.

Parameters:
  • p (int) – The number of positive markers (must be less or equal than the numbers of markers in m
  • m (List of String) – The list of markers
Returns:

The list of positive markers.
Return type:

List of String

 
>>> experience(10,markers(100))
['m16', 'm79', 'm18', 'm13', 'm26', 'm6', 'm11', 'm17', 'm29', 'm77']
experience.markers(p)[source]¶

Returns a list of p markers. Markers are in random ordering.

Parameters:p (int) – The number of markers (must be strictly greater than 0.
Returns:The list of markers
Return type:List of String 
>>> markers(10)
['m9', 'm1', 'm7', 'm2', 'm4', 'm0', 'm6', 'm8', 'm5', 'm3']
